# 02 - 环境搭建与运行指南

## 1. 环境依赖

| 软件 | 最低版本 | 用途 | 安装检查 |
|---|---|---|---|
| JDK | 17+ | 后端运行 | `java -version` |
| Node.js | 18+ | 前端构建 | `node -v` |
| npm | 8+ | 前端包管理 | `npm -v` |
| Docker Desktop | 20+ | 数据库容器 | `docker --version` |
| Git | 2.x | 版本管理 | `git --version` |

> **注意**：本项目后端使用 Maven Wrapper（`mvnw`），无需单独安装 Maven。

---

## 2. 首次搭建步骤

### 2.1 克隆项目

```bash
git clone <repo-url>
cd moveme
```

### 2.2 配置环境变量

复制 `.env` 模板并填写你的配置：

```bash
# .env 文件位于项目根目录，已在 .gitignore 中
# 主要需要修改的配置：
GEMINI_API_KEY=your-actual-gemini-api-key   # LLM 智能推荐需要
```

完整 `.env` 配置项：

```ini
# MySQL（Docker 容器端口）
MYSQL_HOST=localhost
MYSQL_PORT=3307          # 映射到 3307，避免与本地 MySQL 冲突
MYSQL_USER=root
MYSQL_PASSWORD=root123

# Redis（Docker 容器端口）
REDIS_HOST=localhost
REDIS_PORT=6380          # 映射到 6380，避免与本地 Redis 冲突

# JWT
JWT_SECRET=moveme-secret-key-must-be-at-least-256-bits-long-for-hs256-algorithm

# LLM API Keys（填写你有的即可）
GEMINI_API_KEY=
OPENAI_API_KEY=
CLAUDE_API_KEY=
```

### 2.3 启动数据库（Docker）

```bash
cd docker
docker compose up -d
```

> **重置 v2 schema（破坏性）**：当 schema 变了或想从 0 开始，先 `docker compose down -v` 清掉数据卷，再 `up -d` 让 `01-schema.sql` + `02-seed-data.sql` 重新生效。生产慎用。

验证：
```bash
# MySQL 就绪检查
docker exec moveme-mysql mysqladmin ping -u root -proot123 --silent
# 输出: mysqld is alive

# 数据库和表检查
docker exec moveme-mysql mysql -u root -proot123 -e "USE moveme; SHOW TABLES;"
# 应显示 38 张表（v2 schema）

# Redis 就绪检查
docker exec moveme-redis redis-cli ping
# 输出: PONG
```

### 2.4 启动后端

```bash
cd moveme-backend

# 方式一：Maven Wrapper 开发模式（支持热重载）
# Windows:
mvnw.cmd spring-boot:run
# Linux/Mac:
./mvnw spring-boot:run

# 方式二：打包后运行 jar
# Windows:
mvnw.cmd package -DskipTests
# Linux/Mac:
./mvnw package -DskipTests

java -jar target/moveme-backend-1.0.0.jar
```

> **Windows 环境注意**：如果 `JAVA_HOME` 未设置，`mvnw.cmd` 会直接使用 PATH 中的 `java` 命令。
> 如遇到问题，可手动运行：
> ```bash
> java -Dmaven.multiModuleProjectDirectory=. -cp .mvn/wrapper/maven-wrapper.jar org.apache.maven.wrapper.MavenWrapperMain spring-boot:run
> ```

验证后端启动成功：
```bash
curl http://localhost:8080/api/v1/movies/genres
# 应返回: {"code":200,"message":"操作成功","data":[...]}
```

### 2.5 启动前端

```bash
cd moveme-frontend
npm install
npm run dev
```

浏览器访问 http://localhost:5173

### 2.6 种子数据导入（Phase B 完成后）

种子导入由 `SeedAutoRunner` 在后端启动时自动触发（条件：`moveme.seed.auto-import=true` 且 `movies` 表行数 < 50）。

```bash
# 启动后端日志中应看到：
# Top250 seed import started (path=classpath:/seed/top250.json)
# Top250 seed import finished: ok=250 fail=0 persons=1500+ comments=12500+

# 验证导入完成：
docker exec moveme-mysql mysql -uroot -proot123 moveme -e "
  SELECT
    (SELECT COUNT(*) FROM movies)       AS movies,
    (SELECT COUNT(*) FROM movie_top250) AS top250,
    (SELECT COUNT(*) FROM persons)      AS persons,
    (SELECT COUNT(*) FROM movie_comment WHERE source=0) AS douban_comments;
"
# 期望：movies=250, top250=250, persons>=1500, douban_comments>=10000
```

## 3. 端口规划

| 服务 | 端口 | 说明 |
|---|---|---|
| 后端 Spring Boot | 8080 | API 服务 |
| 前端 Vite Dev | 5173 | 开发服务器（代理 /api → 8080） |
| MySQL (Docker) | 3307 | 映射到容器内 3306 |
| Redis (Docker) | 6380 | 映射到容器内 6379 |

> 使用非标准端口（3307、6380）是为了避免与本地已安装的 MySQL/Redis 冲突。

## 4. 开发常用命令

### 4.1 后端

```bash
# 编译
mvnw.cmd compile

# 运行测试
mvnw.cmd test

# 打包（跳过测试）
mvnw.cmd package -DskipTests

# 查看依赖树
mvnw.cmd dependency:tree
```

### 4.2 前端

```bash
# 开发模式
npm run dev

# 构建生产版本
npm run build

# 预览生产版本
npm run preview
```

### 4.3 Docker

```bash
# 启动容器
cd docker && docker compose up -d

# 停止容器
docker compose down

# 查看日志
docker logs moveme-mysql
docker logs moveme-redis

# 进入 MySQL 命令行
docker exec -it moveme-mysql mysql -u root -proot123 moveme

# 进入 Redis 命令行
docker exec -it moveme-redis redis-cli

# 重置数据库（删除数据卷后重启）
docker compose down -v
docker compose up -d
```

## 5. 常见问题

### Q1: `mvnw` 报错 "maven-wrapper.jar 没有主清单属性"

这是因为 maven-wrapper.jar 3.2.0 没有 `Main-Class`，需要用 `-cp` 方式运行：

```bash
java -Dmaven.multiModuleProjectDirectory=. -cp .mvn/wrapper/maven-wrapper.jar org.apache.maven.wrapper.MavenWrapperMain <目标>
```

### Q2: 端口被占用

```bash
# 查看端口占用
netstat -ano | grep :<端口号>

# 修改 .env 中的端口配置
# 同时修改 application.yml 中的默认值
```

### Q3: Docker MySQL 连接被拒绝

等待 MySQL 完全初始化（首次启动约 30 秒）：
```bash
docker exec moveme-mysql mysqladmin ping -u root -proot123 --silent
```

### Q4: 前端页面空白或接口 404

确认 `vite.config.ts` 中的代理配置指向后端地址：
```typescript
proxy: {
    '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
    },
}
```

### Q5: Redis 连接失败

确认 Docker Redis 容器正在运行，且端口映射正确：
```bash
docker ps | grep moveme-redis
# 应看到 0.0.0.0:6380->6379/tcp
```
