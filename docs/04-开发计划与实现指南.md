### 5.1 协同过滤（Item-Based Cosine Similarity）

#### 算法原理

```
1. 构建 用户-电影 评分矩阵

           电影A  电影B  电影C  电影D
   用户1    8      -      7      -
   用户2    9      6      -      8
   用户3    -      7      6      9

2. 计算电影之间的余弦相似度

   sim(A, B) = (A·B) / (|A| × |B|)

   电影A 的评分向量: [8, 9]     （只取有共同评分的用户）
   电影B 的评分向量: [-, 6]

   只有用户2同时评了A和B，数据太少。
   实际中需要至少 2 个共同用户才计算相似度。

3. 为目标用户推荐
   - 找出用户评过高分的电影
   - 找这些电影的最相似电影
   - 过滤已看过的
   - 按相似度加权排序，取 Top-N
```

#### 余弦相似度公式

```
         Σ(Ai × Bi)
sim = ─────────────────
      √Σ(Ai²) × √Σ(Bi²)

### 5.2 内容推荐（Content-Based）

#### 算法原理

```
1. 为每部电影构建特征向量
   - 类型 one-hot: [剧情, 喜剧, 科幻, ...] → [1, 0, 1, 0, ...]
   - 年代 bucket:  [2020s, 2010s, 2000s, ...] → [1, 0, 0, ...]
   - 评分 bucket:  [9+, 8-9, 7-8, ...] → [0, 1, 0, ...]

2. 构建用户偏好向量
   - 对用户评过 >= 7 分的电影的特征向量求平均

3. 计算用户偏好向量与所有未看电影的余弦相似度

4. 返回最相似的 Top-N
```

@Service
public class LlmService {
    private final List<LlmProvider> providers;  // 按优先级排序

    public LlmResponseDTO chat(String systemPrompt, String userPrompt) {
        for (LlmProvider provider : providers) {
            if (!provider.isAvailable()) continue;
            try {
                return provider.chat(systemPrompt, userPrompt);
            } catch (Exception e) {
                log.warn("LLM Provider {} failed, trying next", provider.getName(), e);
            }
        }
        throw new BusinessException(ResultCode.LLM_PROVIDER_UNAVAILABLE);
    }
}
```

    public Object logApi(ProceedingJoinPoint point) throws Throwable {
        long start = System.currentTimeMillis();
        String method = point.getSignature().toShortString();
