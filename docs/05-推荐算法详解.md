## 1. 核心概念

| 概念 | 含义 |
|---|---|
| **特征向量** | 把电影/用户表示为一个数值向量 |
| **相似度** | 两个向量之间的距离/夹角度量 |
| **TF-IDF** | 词频 × 逆文档频率，弱化普遍特征 |
| **余弦相似度** | 向量夹角余弦 |
| **Jaccard** | 集合交并比 |
| **PMI** | 点互信息，比共现 count 更鲁棒 |
| **协同过滤** | 基于用户-物品评分矩阵的相似度 |

## 2. 电影特征向量（MovieFeatureBuilder）

### 2.1 genre_vector (TF-IDF)

每部电影的类型集合是个稀疏特征。简化版直接 one-hot：

```
movie 1: 剧情, 犯罪      → {"剧情": 1, "犯罪": 1}
movie 2: 剧情, 爱情, 喜剧 → {"剧情": 1, "爱情": 1, "喜剧": 1}
```

进化为 TF-IDF（让"剧情"这种普遍类型权重低）：

```
TF(t, d)  = 1 (类型 d 包含 t 则为 1)
DF(t)     = 含类型 t 的电影数
IDF(t)    = log(N / DF(t))           N = 电影总数
TF-IDF(t,d) = TF(t,d) * IDF(t)
```

**Java 草图**：

```java
public class MovieFeatureBuilder {

    public void rebuildAll() {
        List<Movie> movies = movieMapper.selectList(null);
        int N = movies.size();

        // 1. 统计 DF
        Map<Integer, Integer> df = new HashMap<>();   // genre_id → doc_count
        Map<Long, List<Integer>> movieGenreIds = loadMovieGenres();
        for (List<Integer> ids : movieGenreIds.values()) {
            for (Integer gid : ids) df.merge(gid, 1, Integer::sum);
        }

        // 2. 算每部电影的 genre_vector
        Map<Integer, String> genreNames = loadGenreNames();
        for (Movie m : movies) {
            List<Integer> ids = movieGenreIds.getOrDefault(m.getId(), List.of());
            Map<String, Double> vec = new LinkedHashMap<>();
            for (Integer gid : ids) {
                double idf = Math.log((double) N / df.get(gid));
                vec.put(genreNames.get(gid), idf);   // TF=1
            }
            // L2 归一化
            normalizeL2(vec);
            // upsert movie_features.genre_vector
            saveGenreVector(m.getId(), vec);
        }
    }

    static void normalizeL2(Map<String, Double> vec) {
        double sumSq = vec.values().stream().mapToDouble(v -> v * v).sum();
        double norm = Math.sqrt(sumSq);
        if (norm == 0) return;
        vec.replaceAll((k, v) -> v / norm);
    }
}
```

### 2.2 cast_vector

把 (导演 + 编剧 + 主演前 5) 的 person_id 集合做 TF-IDF：

```
TF(p, d)  = 角色权重: director=1.5, lead_actor=1.0, writer=0.5, supporting_actor=0.3
DF(p)     = 该 person 出现在多少部电影
```

为什么要加角色权重：导演主导风格的程度高于第 8 主演。

### 2.3 tag_vector

豆瓣 tags 大多空，先建表占位，后续从详情页爬到再算。算法同 genre。

### 2.4 decade

```java
int decade = (year == null) ? 0 : (year / 10) * 10;
// 1994 → 1990, 2008 → 2000
```

存为 SMALLINT 直接对比 ==。

### 2.5 cluster_id (可选，KMeans)

把 250 部电影聚成 ~15 簇，用于"探索性推荐"（跨簇推用户没见过的口味）。

```
输入：每部电影的 (genre_vector ⊕ cast_vector ⊕ year_normalized) 拼成一个向量
KMeans:
  1. 随机选 K 个中心
  2. 每部电影分配到最近中心 (欧氏距离)
  3. 重新计算每簇中心 = 簇内向量均值
  4. 重复 2-3 直到中心不再变化或迭代 50 次
输出：每部电影的 cluster_id (0..K-1)
```

K=15，迭代 50 次，250 部规模下毫秒级。可不实现。

## 3. 相似度算法

### 3.1 余弦相似度

```
cosine(A, B) = (A · B) / (|A| * |B|)
             = sum(A[i] * B[i]) / (sqrt(sum(A[i]^2)) * sqrt(sum(B[i]^2)))
```

值域 [-1, 1]，对正向量是 [0, 1]。1 = 方向完全一致，0 = 正交。

```java
public static double cosine(Map<String, Double> a, Map<String, Double> b) {
    Set<String> keys = new HashSet<>(a.keySet());
    keys.retainAll(b.keySet());
    double dot = 0;
    for (String k : keys) dot += a.get(k) * b.get(k);
    double normA = Math.sqrt(a.values().stream().mapToDouble(v -> v*v).sum());
    double normB = Math.sqrt(b.values().stream().mapToDouble(v -> v*v).sum());
    return (normA == 0 || normB == 0) ? 0 : dot / (normA * normB);
}
```

向量已 L2 归一化时 `normA == normB == 1`，省掉除法。

### 3.2 Jaccard 相似度（集合）

```
jaccard(A, B) = |A ∩ B| / |A ∪ B|
```

适合无权重的集合特征（国家/语言）：

```java
static double jaccard(Set<?> a, Set<?> b) {
    if (a.isEmpty() && b.isEmpty()) return 0;
    Set<Object> intersection = new HashSet<>(a);
    intersection.retainAll(b);
    Set<Object> union = new HashSet<>(a);
    union.addAll(b);
    return (double) intersection.size() / union.size();
}
```

## 4. ContentSimilarityCalculator

### 4.1 加权融合公式

```
content_sim(A, B) = 0.45 * cosine(genre_vec_A, genre_vec_B)
                  + 0.30 * cosine(cast_vec_A,  cast_vec_B)
                  + 0.10 * (decade_A == decade_B ? 1 : 0)
                  + 0.10 * jaccard(country_A, country_B)
                  + 0.05 * (sameLanguage ? 1 : 0)
```

权重和 = 1。可调。

### 4.2 实现思路

```java
public class ContentSimilarityCalculator {

    @Value("${moveme.recommendation.similarity.top-k:50}")
    private int topK;

    public void rebuildAll() {
        List<Long> movieIds = movieMapper.selectAllIds();
        int N = movieIds.size();

        // 1. 一次性加载所有 features 进内存（250 部毫无压力）
        Map<Long, MovieFeatureSnapshot> features = loadAllFeatures(movieIds);

        // 2. 250×250 = 62500 对，遍历
        Map<Long, List<SimilarityRow>> simByA = new HashMap<>();
        for (int i = 0; i < N; i++) {
            for (int j = 0; j < N; j++) {
                if (i == j) continue;
                Long a = movieIds.get(i), b = movieIds.get(j);
                double s = computeSim(features.get(a), features.get(b));
                if (s < 0.05) continue;   // 阈值过滤
                simByA.computeIfAbsent(a, k -> new ArrayList<>())
                      .add(new SimilarityRow(a, b, s));
            }
        }

        // 3. 每个 a 取 top-K，rank_no 1..K
        movieSimilarityMapper.deleteByAlgorithm("CONTENT");
        for (Map.Entry<Long, List<SimilarityRow>> e : simByA.entrySet()) {
            List<SimilarityRow> top = e.getValue().stream()
                .sorted((x,y) -> Double.compare(y.score, x.score))
                .limit(topK)
                .toList();
            int rank = 1;
            for (SimilarityRow r : top) {
                r.algorithm = "CONTENT";
                r.rankNo = rank++;
                movieSimilarityMapper.insert(r);
            }
        }
    }
}
```

**性能**：250 部 → 62500 对 × 每对几百纳秒 ≈ 几十毫秒。
未来量上去（10K 部 → 1 亿对）需要 LSH 或分块 + 倒排索引（按 genre 先桶分）。

## 5. CFItemSimilarityCalculator（协同过滤）

### 5.1 评分矩阵

```
        movieA  movieB  movieC
user1     5       4      -
user2     4       5      3
user3     -       3      4
```

`-` 表示没评分。

### 5.2 余弦（Item-Based）

电影 a 的"用户向量" = 所有给 a 评过分的用户的评分。比较 a 和 b 的相似度：

```
对所有"既评了 a 又评了 b"的用户 U_ab:
  cf_sim(a,b) = cosine(rating_a[U_ab], rating_b[U_ab])
```

**重要**：只用共同用户的评分（不用 0 填充缺失），否则相似度被人气稀释。

### 5.3 数据稀疏阈值

```
if |U_ab| < min_co_users (默认 5):  skip
```

阈值小时噪声大；阈值大时召回率低。冷启动期可以调到 1。

### 5.4 把 favorites 当弱评分

```
status=1 (已看): score = 7  权重 0.5
status=0 (想看): 不计入 CF
```

合并到 ratings 矩阵里参与计算。

### 5.5 实现要点

```java
// 加载评分矩阵
// movieRatings: movie_id → Map<user_id, score>
Map<Long, Map<Long, Double>> movieRatings = loadRatings();

// 对每对电影
for (Long a : movieRatings.keySet()) {
    for (Long b : movieRatings.keySet()) {
        if (a >= b) continue;   // 对称，只算上三角
        Map<Long,Double> ra = movieRatings.get(a);
        Map<Long,Double> rb = movieRatings.get(b);
        Set<Long> common = new HashSet<>(ra.keySet());
        common.retainAll(rb.keySet());
        if (common.size() < minCoUsers) continue;
        double dot = 0, na = 0, nb = 0;
        for (Long u : common) {
            double xa = ra.get(u), xb = rb.get(u);
            dot += xa * xb; na += xa*xa; nb += xb*xb;
        }
        double sim = dot / (Math.sqrt(na) * Math.sqrt(nb));
        // 写 (a,b) 和 (b,a) 各一行（对称）
    }
}
```

## 6. CoOccurrenceCalculator（共现 + PMI）

### 6.1 共现 count

```
co_count(a, b) = COUNT(DISTINCT u) WHERE
   user u 喜欢 a (rating>=7 OR favorite.status=1) AND
   user u 喜欢 b
```

### 6.2 PMI（点互信息）

```
p(a)   = (喜欢 a 的用户数) / (总用户数)
p(b)   = (喜欢 b 的用户数) / (总用户数)
p(a,b) = co_count(a,b) / 总用户数

pmi(a,b) = log(p(a,b) / (p(a) * p(b)))
```

PMI > 0 表示比独立预期更频繁共现；< 0 表示反相关；0 表示独立。

PMI 比纯 count 鲁棒在哪：纯 count 偏向流行电影（《肖申克》和啥都共现高），PMI 把"流行的独立性"扣掉。

### 6.3 阈值

```
co_count >= 2     // 至少 2 个共同用户才入库
```

冷启动期 ratings 几乎为空，整张表会很空，没关系。

## 7. UserFeatureBuilder

### 7.1 加权聚合

```
对用户 u 的每条行为 (movie_id, weight):
  user.genre_vector += weight * movie.genre_vector
  user.cast_vector  += weight * movie.cast_vector

行为权重:
  rating score >= 8:  weight = +1.0
  rating score 7:     weight = +0.5
  rating score 5-6:   weight = 0
  rating score <= 4:  weight = -0.5
  favorite status=1 (已看): weight = +0.6
  favorite status=0 (想看): weight = +0.3
  view_history: weight = +0.2
```

**最后归一化**：每个向量 L2 归一化。

### 7.2 user_genre_pref / user_person_pref

`user_features.genre_vector` 是 JSON 不便索引。把 top-N 拍平到关联表：

```java
// 把 user.genre_vector 里 score > 0 的项写入 user_genre_pref

    userGenrePrefMapper.upsert(userId, gid, e.getValue());
}
```

## 8. PopularityScorer / QualityScorer / FreshnessScorer

### 8.1 Popularity（热度）

```
popularity = log10(douban_votes + 1) * douban_rating * decay(now - release_date)

decay(age_years) = 0.5 ^ (age_years / 5)   // 半衰期 5 年
```

老电影（80 年代）的人气在分数上自然衰减，给新电影机会。

### 8.2 Quality（质量）

```
quality = douban_rating
        + 0.3 * (won_oscar_best_picture ? 1 : 0)
        + 0.1 * (top250_rank ? (251 - rank) / 250 : 0)
        + 0.2 * (won_any_major_award ? 1 : 0)
```

### 8.3 Freshness（新鲜度）

```
freshness = exp(- (now - release_date) / 3y)
```

3 年前的电影 freshness ≈ 0.37，10 年前 ≈ 0.04。给"最新上映"页用。

### 8.4 Java

```java
public void rescoreAll() {
    LocalDate now = LocalDate.now();
    List<Movie> movies = movieMapper.selectList(null);
    for (Movie m : movies) {
        Double pop = popularity(m, now);
        Double qual = quality(m);
        Double fresh = freshness(m, now);
        movieMapper.updateScores(m.getId(), pop, qual, fresh);
    }
}

double popularity(Movie m, LocalDate now) {
    if (m.getDoubanVotes() == null || m.getDoubanRating() == null) return 0;
    double base = Math.log10(m.getDoubanVotes() + 1) * m.getDoubanRating();
    double ageYears = (m.getReleaseDate() == null) ? 0
        : ChronoUnit.DAYS.between(m.getReleaseDate(), now) / 365.0;
    double decay = Math.pow(0.5, ageYears / 5.0);
    return base * decay;
}
```

## 9. 推荐召回 + 排序（在线推理）

```
1. 召回 (recall) ─ 给用户拼候选池 (size ~200)
   ├─ 用户最近高分电影 → 取 movie_similarity (CONTENT) top-20 each
   ├─ 用户最近高分电影 → 取 movie_similarity (CF_ITEM) top-20 each
   ├─ 用户最爱类型 → movies WHERE genre IN ... ORDER BY popularity DESC LIMIT 30
   └─ 全局 top-popular 兜底 LIMIT 20

2. 过滤
   ├─ 已看过的 (ratings 或 favorites status=1) 全部排除
   ├─ status != 1 的电影排除
   └─ 用户黑名单类型排除（如果未来支持）

3. 排序 (rank)
   final_score = 0.30*content_score + 0.50*cf_score + 0.20*llm_score + 0.10*popularity_bonus
   多样性约束：top-30 至少 3 种主类型

4. 写缓存 + 返回 top-20
```

## 10. 复杂度小结

| 算法 | 250 部规模 | 10K 部规模 |
|---|---|---|
| MovieFeatureBuilder | < 50ms | ~1s |
| ContentSimilarity (250×250) | ~50ms | 10K×10K = 1 亿对，需要分块/LSH，~10s |
| CFItemSimilarity | 取决于评分密度，初期 < 100ms | 大量评分时几秒 |
| CoOccurrence | < 100ms | 几秒 |
| UserFeatureBuilder (单用户) | < 5ms | < 50ms |
| 在线推理 | < 50ms（命中缓存 < 5ms） | < 200ms |
