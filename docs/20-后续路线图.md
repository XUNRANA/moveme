## Phase D — RecommendationService（在线推理）

**目标**：消费 Phase C 预计算的特征/相似度，实时给用户出推荐。

### 包结构
```
module/recommendation/strategy/
├── RecommendationStrategy.java           # 接口
├── ColdStartRecommender.java             # 未登录 / 0 行为
├── ContentBasedRecommender.java
├── ItemBasedCFRecommender.java
└── HybridRecommender.java                # 融合
service/
├── RecommendationService.java            # 路由 + 缓存 + 解释
└── impl/RecommendationServiceImpl.java
```

### 关键流程
```
RecommendationService.recommend(userId, n):
  1. 查 user_reco_cache，命中且未过期 → 返回
  2. 加载用户状态（rating_count, favorite_count）
  3. 路由策略：
       未登录 / 0 行为 → ColdStartRecommender
       行为 < 5         → ContentBasedRecommender
       行为 ≥ 5         → HybridRecommender
  4. 召回 + 排序 + 多样性约束
  5. 写 user_reco_cache
  6. 返回前端
```

详见 [04 - 推荐系统总体设计](./04-推荐系统总体设计.md) 第 5-6 节。

## Phase F — LLM Recommender 接入

### LLMService 抽象
```
LLMProvider (interface)
├── GeminiProvider
├── ClaudeProvider
└── OpenAIProvider

LLMService:
  - 责任链降级：Gemini → Claude → OpenAI（按 API key 是否配置）
  - 工厂模式：根据 application.yml 选 primary
```

### Prompt 模板
```
你是电影推荐专家。用户偏好画像：
  - 最爱类型: {top_3_genres}
  - 最爱导演: {top_3_directors}
  - 最近看过高分: [movie_id_1, movie_id_2, ...]
  - 平均打分倾向: {avg_rating}

候选电影列表（已过滤已看）:
[{id, title, genres, director, year, rating}, ...]

请从候选里选出 {n} 部最适合该用户的电影，给出 JSON：
[{"movie_id": 123, "reason": "因为...", "score": 0.85}, ...]
```

### 落点
- 写 `llm_reco_log` 记录 prompt + response + cost
- LLM 结果的 movie_id 必须在候选列表里（不能让 LLM 杜撰）
- 失败/超时降级走 Hybrid 不带 LLM

## Phase G — 前端展示

### 路由
```
/                    首页（热门 + 推荐 + Top250）
/movies              浏览（筛选/排序/分页）
/movies/:id          详情（5 星分布 / 短评 / 演职员 / 相关电影 / 奖项）
/recommendations     我的推荐（含解释）
/login /register     用户认证
/me                  我的页面
/me/favorites        我的收藏（想看/已看 tab）
/me/ratings          我的评分
/me/history          浏览历史
/search?q=...        搜索结果
```

### 关键组件
- `<MovieCard>` 卡片（海报 + 评分 + 类型标签）
- `<RatingBreakdown>` 5 星分布柱状图
- `<RecommendationReason>` 推荐解释（"因为你看过 X"）
- `<CommentList>` 短评列表（FULLTEXT 搜索 + 排序）
- `<RelatedMovies>` 详情页底部相关电影（来自 movie_related + Phase C 相似度）

### 状态管理
- `userStore` (Pinia): JWT 状态 + 用户信息
- `movieStore`: 电影详情缓存（避免重复请求）
- `recommendationStore`: 推荐结果缓存

## Phase H — 评估 & 优化

### 埋点
- `recommendation_view` 事件：用户看到推荐
- `recommendation_click` 事件：用户点击哪部
- 计算 CTR = clicks / views

### A/B 测试
- 把策略权重做成配置：`hybrid.weights.{content, cf, llm}`
- 同一用户分桶到不同权重组合，比较 CTR

### 性能
- 在线推理 P95 < 200ms
- 慢查询日志监控（开 mysql slow_query_log）
- 推荐结果命中缓存率 > 80%
