## 1. 立刻可做的优化（小而有价值）

### 1.1 推荐离线计算默认启用
当前 `application.yml`：
```yaml
moveme:
  recommendation:
    enabled: false
    bootstrap-on-start: false
```
打开后能跑通：MovieFeatureBuilder → ContentSimilarityCalculator → CFItemSimilarityCalculator → CoOccurrenceCalculator → UserFeatureBuilder。

需要做：先按 `41-推荐特征计算.md` 跑一次 dry-run，确认 250 部规模下计算时间在分钟级。

### 1.2 推荐结果缓存主动失效
`user_reco_cache` TTL 24h 已就位，但用户写新评分/收藏时不主动 evict。补一个事件监听：
```java
@EventListener(RatingChangedEvent.class)
public void invalidate(RatingChangedEvent e) { ... }
```

### 1.3 评分 → movie_comment 同步
`ratings.comment` 与 `movie_comment(source=1)` 现在是两份数据。后续要么单向同步、要么 UI 上拆开两套录入。

### 1.4 搜索热词榜
`search_history` 已记录全量搜索（含匿名），可做：
- 首页"近 24h 热搜"小模块
- 管理后台运营页"本周搜索词云"

## 2. Phase E：A/B 测试与运营工具

### 2.1 推荐策略 A/B
- 把 HYBRID 融合权重做成可配置：`hybrid.weights.{content, cf, llm}`
- 用户 ID hash 分桶到不同权重组合
- 写 `recommendation_logs.strategy_type` 区分桶号
- 管理后台加"按桶看 CTR"图表

### 2.2 LLM Recommender 接入
- LLMService 抽象：`GeminiProvider` → `ClaudeProvider` → `OpenAIProvider`（按配置降级）
- `application.yml` 的 `moveme.llm.*` 已就位（API key 占位）
- 落点：`llm_reco_log` 记录 prompt / response / cost
- 失败/超时降级走 Hybrid 不带 LLM

### 2.3 运营埋点
- `recommendation_view` / `recommendation_click` 事件
- CTR = clicks / views，按策略与时段切片
- 慢查询日志 + 推荐缓存命中率监控

## 3. Phase F：向量化与新算法

### 3.1 文本 embedding
- 用 LLM API 或 sentence-transformer 把 `movies.summary` 转 32~768 维 dense
- 落 `movie_features.embedding` JSON 列
- ContentSimilarityCalculator 增加一个 EMBEDDING 算法（与 CONTENT/CF_ITEM 并列）

### 3.2 人物 / 类型 embedding
- 把 `cast_vector` / `genre_vector` 用 word2vec 风格学到 embedding
- 解决 one-hot 稀疏问题

### 3.3 检索升级
- MySQL FULLTEXT + ngram 顶不住时切 Elasticsearch
- 加 "类型 + 年代 + 评分" facet 筛选

## 4. Phase G：内容扩展

### 4.1 多榜单体系
- 当前只有 Top250 + 豆瓣 chart-genres + boards + annual
- 扩展模式：`movie_list` 表 + `movie_list_membership(movie_id, list_id, rank_no)`
- 候选清单：华语 Top100、IMDb Top250、Netflix Originals、年度十佳

### 4.2 短剧集 / 综艺
- schema 已用 `movies` 抽象，加 `media_type` 列即可（电影/剧集/综艺）
- 爬虫需要扩展豆瓣的 tv subject 解析

### 4.3 视频试映片段
- 不持有版权，只挂播放链接（`movie_play_link` 已就位）

## 6. 工程化欠债（与业务无关但应做）

- **测试覆盖**：当前测试稀薄；至少给 SeedImportService、UserService、MovieService 加单元 + 集成测试
- **CI/CD**：GitHub Actions 跑 `mvnw test` + `npm run build`，产出 jar/dist artifact
- **生产部署**：Nginx 反代 + HTTPS + 后端多实例 + 日志收集
- **慢查询**：开 mysql slow_query_log，把 > 500ms 的 SQL 全部跑 EXPLAIN
- **指标暴露**：Spring Boot Actuator + Prometheus + Grafana
