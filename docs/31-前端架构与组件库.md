### 路由守卫（`router/index.ts`）

```ts
beforeEach(async (to, _from, next) => {
  const userStore = useUserStore()
  // 有 token 但 userInfo 未水合 → 自动拉取（刷新页面后场景）
  if (userStore.isLoggedIn && !userStore.userInfo) {
    try { await userStore.fetchUserInfo() } catch { userStore.logout() }
  }
  if (to.meta.requiresAuth && !userStore.isLoggedIn) next('/login')
  else if (to.meta.requiresAdmin && !userStore.isAdmin) next('/')
  else next()
})
```

## 4. 状态管理（Pinia · `stores/user.ts`）

```ts
export interface UserInfo {
  id: number
  username: string
  email: string
  nickname: string
  avatarUrl: string
  bio?: string
  role: number        // 0=普通 / 1=管理员
  createdAt?: string
}
```

| 字段/方法 | 说明 |
|---|---|
| `accessToken` | ref，localStorage 持久化 |
| `refreshToken` | ref，localStorage 持久化 |
| `userInfo` | ref<UserInfo \| null> |
| `isLoggedIn` | computed: `!!accessToken` |
| `isAdmin` | computed: `userInfo?.role === 1` |
| `login(u, p)` | POST `/auth/login` → 存 token + 自动 fetchUserInfo |
| `register(u, p, email?, nick?)` | POST `/auth/register` |
| `fetchUserInfo()` | GET `/users/me` |
| `logout()` | 清 token + 清 localStorage |

> 当前只有 `user.ts` 一个 store。movieStore / recommendationStore 等是路线图项，没必要预设而是按出现需求再添加。

### 用法建议

- **颜色**：原色尽量走 token，避免组件里写死十六进制；想做高对比新主题时只改 `variables.css`
- **圆角**：按尺寸语义（sm 用于 badge、md 用于 card、lg 用于 hero、pill 用于 chip）
- **阴影**：默认 card；hover 时切 hover；强调时叠 glow
- **动效**：transition 一律走 `--ease-out + --duration-normal`，保持节奏一致

## 9. 自定义指令 `v-animate`（`directives/vAnimate.ts`）

```vue
<div v-animate>普通元素，进入视口时 fade-up</div>
<div v-animate="'slide-up'">指定动画类型</div>
```

实现要点（仅 24 行）：
- 单例 `IntersectionObserver`（threshold 0.1，`rootMargin: '0px 0px -40px 0px'`）
- `mounted` 时挂 `.animate-on-scroll` + `.anim-{type}`，加入观察队列
- 首次进入视口添加 `.is-visible`，触发 CSS 过渡；之后 `unobserve` 让元素不再被监控

为什么手写而非引第三方：项目体量小，单一指令 24 行 vs 引一个动画库（运行时 5KB+），手写更克制。

## 10. 海报防盗链策略

豆瓣海报 URL 带 Referer 防盗链，浏览器 `<img src="https://img1.doubanio.com/...">` 直接加载 403。

后端在导入阶段就把图片本地化（`data/posters/`），通过 `/static/posters/` 暴露。前端 `resolvePoster()`：

```ts
function resolvePoster(movie) {
  return movie?.posterLocalPath || movie?.posterUrl || undefined
}
```

`PosterImage.vue` 内部已经 wrap 了这个逻辑 + 骨架屏 + 加载失败占位，组件内尽量用 `<PosterImage :movie="m" />` 而不是裸 `<img>`。

## 11. 类型安全约定

- API 调用方加 `as ApiEnvelope<X>` 显式断言，让调用方拿到精确类型而不是 `any`
- `MovieItem` / `PersonDetail` 等核心类型集中在 `api/movies.ts`，组件 props 直接复用，避免类型漂移
- `vue-tsc -b` 在 `npm run build` 时强制类型检查，类型错误会断 CI

## 12. 常见前端调试

| 现象 | 排查 |
|---|---|
| 接口 401 | `localStorage.accessToken` 是否存在；是否过期；axios 拦截器是否注入头 |
| 海报全 403 | 用了 `posterUrl` 而非 `posterLocalPath`；确认 `<PosterImage>` 或 `resolvePoster()` 是否调用 |
| 路由守卫死循环 | `fetchUserInfo()` 抛错没 catch，会一直 logout 又拉 |
| Tailwind class 不生效 | `vite.config.ts` 是否启用 `@tailwindcss/vite` 插件；`global.css` 顶部是否 `@import "tailwindcss"` |
| HMR 不工作（WSL2/Docker） | 改 `vite.config.ts` 的 `server.watch.usePolling: true` |
