# 32 - 管理后台使用指南

> 入口：以管理员账号登录前端后访问 `/admin/dashboard`（路由 `requiresAuth + requiresAdmin`）。
>
> 后端接口：`/api/v1/admin/**`（`SecurityConfig` 限 `ROLE_ADMIN`）。
>
> 数据范式：所有接口走 `Result<T>` 信封；分页走 `IPage<T>`（参数 `page`/`size`，默认 1/10）。

---

## 0. 准备工作

### 0.1 拥有管理员账号

两种路径：

1. 使用 `docker/mysql/init/02-seed-data.sql` 内置的 `admin` 账号（默认密码见种子脚本，建议首次登录后改）。
2. 把已注册的普通账号升级：

```sql
docker exec moveme-mysql mysql -uroot -proot123 moveme -e "
  UPDATE users SET role = 1 WHERE username = 'YOUR_USERNAME';
"
```

升级后该用户需重新登录前端，token 重新签发后 `userInfo.role` 才会变为 1。

### 0.2 进入后台

前端登录 → 顶部导航右侧用户菜单 → "管理后台"（仅 `isAdmin` 时可见）→ 进入 `/admin/dashboard`。

如果直接访问 `/admin/dashboard` 而非管理员，路由守卫会 `next('/')` 把你弹回首页。

## 1. AdminDashboard 页面概览

页面分为 6 个区块，每个区块对应一个组件：

```
┌─────────────────────────────────────────────────────────────┐
│  系统总览（AdminSystemOverview）                              │
│  ├─ 各模块数据规模简报                                        │
├─────────────────────────────────────────────────────────────┤
│  6 张统计卡（AdminStatsCards）                                │
│  movieCount | userCount | todayNewUsers | ratingCount        │
│  lastCrawlStatus + lastCrawlTime | recoLogCount              │
├─────────────────────────────────────────────────────────────┤
│  用户管理表（AdminUserTable）                                  │
│  关键字 / 角色 / 状态 筛选 + 分页 + 行内角色/状态切换           │
├─────────────────────────────────────────────────────────────┤
│  爬虫面板（AdminCrawlerPanel）                                │
│  选 taskType + 填 params → 一键 trigger                       │
├─────────────────────────────────────────────────────────────┤
│  数据导入日志（AdminImportLogs）                              │
│  源文件 / 总数 / 成功 / 失败 / 时间                            │
├─────────────────────────────────────────────────────────────┤

│  用户 / 策略 / LLM 厂商 / 耗时                                │
└─────────────────────────────────────────────────────────────┘
```

## 2. 系统统计（AdminStatsCards）

调用：`GET /api/v1/admin/stats` → `AdminStatsVO`。

| 卡片 | 字段 | 含义 |
|---|---|---|
| 电影总数 | `movieCount` | `movies` 表行数 |
| 用户总数 | `userCount` | `users` 表行数 |
| 今日新增 | `todayNewUsers` | `users.created_at >= 当天 00:00` |
| 评分总数 | `ratingCount` | `ratings` 表行数 |
| 爬虫状态 | `lastCrawlStatus` + `lastCrawlTime` | `crawl_logs` 最近一条 |
| 推荐日志 | `recoLogCount` | `recommendation_logs` 行数（默认 0，开启推荐后增长） |

总览组件 `AdminSystemOverview` 复用同一个 stats 接口，额外展示 favoriteCount / viewHistoryCount / searchHistoryCount / personCount / genreCount / crawlLogCount / importLogCount / movieCommentCount。

## 3. 用户管理（AdminUserTable）

调用：`GET /api/v1/admin/users?keyword=&role=&status=&page=&size=`。

### 3.1 过滤
- `keyword` 模糊匹配 `username / nickname / email`
- `role` 0=普通 / 1=管理员
- `status` 0=禁用 / 1=启用

### 3.2 行内操作

| 操作 | 接口 | 触发约束 |
|---|---|---|
| 切换状态（禁用/启用） | `PUT /admin/users/{id}/status` Body `{status: 0\|1}` | 不能禁用自己 |
| 切换角色（普通/管理员） | `PUT /admin/users/{id}/role` Body `{role: 0\|1}` | 不能把自己降级 |

### 3.3 典型场景

- **批量审查异常账号**：filter `status=0` 列出所有被禁用账号，按时间排查
- **晋升运营**：先让对方注册普通账号，再用此页一键升级 role=1
- **应急封禁**：发现刷评的账号，立即把 status 切 0；其 JWT 仍有效直到过期，需要立刻踢可手动 `DEL auth:refresh_token:{userId}` + 等 access 过期（最坏 2h）

## 4. 爬虫面板（AdminCrawlerPanel）

接口：
- `POST /api/v1/admin/crawler/trigger`（通用触发）
- `POST /api/v1/admin/crawler/movie/{subjectId}`（单部电影快捷）
- `GET /api/v1/admin/crawler/status`（状态查询）

### 4.1 任务类型 `taskType`

| 值 | 用途 | 典型 params |
|---|---|---|
| `CHART` | 抓豆瓣分类排行榜 | `{ "genre": "剧情" }` |
| `COMMENTS` | 补抓指定电影/全量电影的短评 | `{ "movieId": 123 }` 或省略表示全量 |
| `ANNUAL` | 抓年度榜 | `{ "year": 2025 }` |
| `ENRICH` | 增强抓取（详情字段补全） | `{}` |
| `SINGLE` | 抓单部豆瓣 subject | `{ "subjectId": "1292052" }` |

后端会在 `crawl_logs` 写一条 `status=running` 的记录，触发 Python 子进程，结束后更新 `success_count / fail_count / status / finished_at`。

### 4.2 何时手动触发

- **定时 cron 之外的应急**：豆瓣临时上新榜，等不及周一 03:00
- **导入失败后重跑**：参考 `AdminImportLogs` 找到失败 importId，先 truncate 关联表，再触发 `SeedAutoRunner`（如果改用爬虫则走 `CHART`）
- **加新电影**：知道豆瓣 ID 就走 `SINGLE`，5-10 秒入库

### 4.3 风险

- Python 进程同时多份会撞豆瓣的反爬。`application.yml` 已经设 `lock-timeout-minutes: 120`，超时自动释放锁，但仍建议人工避免并发。
- 频繁触发可能导致豆瓣返回风控页（HTML 含 "sec.douban.com"），那种情况 `crawl_logs.error_message` 会写 "blocked"，需要换 IP/cookie。

## 5. 数据导入日志（AdminImportLogs）

调用：`GET /api/v1/admin/import-logs?page=&size=`。

| 字段 | 含义 |
|---|---|
| source | "top250.json" / "crawler" / "manual" |
| filePath | 导入文件的绝对/相对路径 |
| moviesTotal / moviesOk / moviesFail | 计数 |
| personsOk / commentsOk | 关联表入库数 |
| errors | JSON，失败明细（前 N 条） |
| startedAt / finishedAt | 时间窗口 |

> 用途：定位一次性导入是否成功；失败时通过 `errors` 快速定位字段映射问题。导入逻辑细节见 [40-种子导入器](./40-种子导入器.md) 与 [历史-Phase-B-业务逻辑细节](./历史-Phase-B-业务逻辑细节.md)。

## 6. 推荐日志（AdminRecoLogs）

调用：`GET /api/v1/admin/reco-logs?page=&size=`。

| 字段 | 含义 |
|---|---|
| userId | 推荐请求用户 |
| strategyType | `CONTENT` / `CF_ITEM` / `HYBRID` / `COLD_START` / `LLM` |
| llmProvider | 走 LLM 时记录是 Gemini/Claude/OpenAI |
| latencyMs | 端到端耗时 |
| createdAt | 触发时间 |

> 在线推理（Phase E）未实现前该表无数据。开启后用此页观察策略分布与延迟分布，作为 A/B 测试基础。

## 7. 安全约束总结

- 所有 `/api/v1/admin/**` 必须带 admin 的 JWT，否则 403
- 失效的 token（过期 / 用户被禁用 / token revoked）→ 401，前端拦截器自动 logout
- 跨域已在 `WebMvcConfig` 配置允许 `/api/**`，但只对开发期开放；生产环境应收紧 `allowedOrigins`

## 8. 常见操作脚本

```bash
# 后台一键看健康
curl -H "Authorization: Bearer <admin_token>" \
  http://localhost:8080/api/v1/admin/stats | jq

# 触发剧情榜爬取
curl -X POST -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{"taskType":"CHART","params":{"genre":"剧情"}}' \
  http://localhost:8080/api/v1/admin/crawler/trigger

# 看最近 5 次爬虫
curl -H "Authorization: Bearer <admin_token>" \
  "http://localhost:8080/api/v1/admin/crawl-logs?page=1&size=5" | jq

# 把用户 7 升为管理员
curl -X PUT -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{"role":1}' \
  http://localhost:8080/api/v1/admin/users/7/role
```
