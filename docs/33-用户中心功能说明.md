# 33 - 用户中心功能说明

> 入口：登录后访问 `/user/dashboard`（路由 `requiresAuth`）。
>
> 后端接口：`/api/v1/users/me/**`（需登录，`userId` 自动从 JWT 取）。
>
> 页面由 6 个组件拼出：`UserProfileCard` + `UserStatsBar` + `UserRatingsList` + `UserFavoritesGrid` + `UserHistoryTimeline` + `UserTasteProfile`。

---

## 0. 数据来源

| 区块 | 接口 | 主要表 |
|---|---|---|
| 档案 | `GET /users/me` / `PUT /users/me` / `POST /users/me/avatar` | `users` |
| 统计 | `GET /users/me/stats` | `ratings` / `favorites` / `view_history` 聚合 |
| 评分 | `GET /users/me/ratings` | `ratings` |
| 收藏 | `GET /users/me/favorites` | `favorites` |
| 历史 | `GET /users/me/history` | `view_history` |
| 口味 | `GET /users/me/taste` | `user_genre_pref` / `user_person_pref` |

---

## 1. 个人档案（UserProfileCard）

### 1.1 展示
- 头像（圆形，默认占位 → 加载用户头像）
- 用户名（不可编辑）
- 昵称（可编辑）
- 邮箱（可编辑）
- 简介 `bio`（可空）
- 加入时间 `createdAt`

### 1.2 头像上传
- 点击头像 → 文件选择
- `POST /users/me/avatar`（`multipart/form-data`，字段名 `file`）
- 上限 10MB，类型 jpg/png
- 后端落 `./data/avatars/users/{userId}.{ext}`，url 通过 `/static/avatars/users/...` 返回
- 前端立即用返回的 url 更新 `userStore.userInfo.avatarUrl`

### 1.3 昵称/邮箱修改
- `PUT /users/me` Body `{ nickname, email }` —— 仅这两个字段
- 不动用户名（用户名一旦注册不允许改，避免主键飘移）
- 不动密码（密码改逻辑独立，建议未来加 `/users/me/password` 接口）

## 2. 统计条（UserStatsBar）

调用：`GET /users/me/stats` → `UserStatsVO`：

```json
{
  "ratingCount":  12,
  "wishCount":    5,    // favorites where status=0
  "watchedCount": 18,   // favorites where status=1
  "historyCount": 34    // view_history 条数（按浏览次数堆积）
}
```

四个数据卡。点击可作为 tab 切换器，过滤下方"收藏网格"按 status。

---

## 3. 评分列表（UserRatingsList）

调用：`GET /users/me/ratings?page=1&size=10` → `IPage<UserRatingItem>`：

```json
{
  "id": 1, "movieId": 42,
  "title": "肖申克的救赎",
  "posterUrl": "...",
  "posterLocalPath": "/static/posters/1292052.jpg",
  "score": 10,
  "comment": "希望让人自由",
  "createdAt": "2026-03-12 22:00:00"
}
```

UI：
- 每条左侧海报 + 右侧"分数 + 评论 + 时间"
- 海报可点击跳详情 `/movies/:id`
- 分页：底部翻页器

> 注：站内评分（`ratings` 表）与站内评论（`movie_comment` source=1）目前是两个动作。详情页上的"评分"会写 `ratings`，"短评"会写 `movie_comment`，两者独立持久化。后续优化方向见 [20-后续路线图](./20-后续路线图.md) 第 1.3 节。

## 4. 收藏网格（UserFavoritesGrid）

调用：`GET /users/me/favorites?status=&page=1&size=12` → `IPage<UserFavoriteItem>`：

```json
{
  "id": 1, "movieId": 42, "title": "肖申克的救赎",
  "posterUrl": "...", "posterLocalPath": "/static/posters/1292052.jpg",
  "doubanRating": 9.7, "year": 1994,
  "genres": ["剧情","犯罪"],
  "status": 0,           // 0=想看 / 1=已看
  "createdAt": "2026-03-12 21:00:00"
}
```

UI：
- 顶部 Tab：全部 / 想看 / 已看
- 网格响应式（6→3→2 列）
- 卡片：海报 + 标题 + 评分 + 年份 + 状态徽章
- 卡片右下角"取消收藏"按钮 → `DELETE /users/me/favorites?movieId=`

### 收藏状态切换流程

详情页里：
1. 进入电影详情 → `GET /users/me/favorites/check?movieId=` 拿到 `FavoriteStatus` 或 `null`
2. 用户点"想看" / "看过"：
   - 已收藏且状态相同 → 提示已收藏
   - 已收藏但状态不同 → 再 POST 一次覆盖
   - 未收藏 → POST `/users/me/favorites`

## 5. 历史时间线（UserHistoryTimeline）

调用：`GET /users/me/history?page=1&size=20` → `IPage<UserHistoryItem>`：

```json
{
  "id": 1, "movieId": 42,
  "title": "肖申克的救赎",
  "posterUrl": "...",
  "viewedAt": "2026-03-12 22:00:00"
}
```

UI：
- 按 `viewedAt` 倒序
- 按日期分组（今天 / 昨天 / 近 7 天 / 更早）
- 每条小卡：海报缩略 + 标题 + 浏览时间

### 数据来源副作用
- 前端 `MovieDetail.vue` mounted 时调 `POST /users/me/history { movieId }`
- 后端不去重，按时间堆积；以"每次访问都记一行"为约定
- 若用户匿名浏览（无 token），不会写记录

## 6. 口味档案（UserTasteProfile）

调用：`GET /users/me/taste` → `UserTasteVO`：

```json
{
  "genrePrefs": [
    { "genreId": 3, "genreName": "剧情", "score": 0.82 },
    { "genreId": 5, "genreName": "犯罪", "score": 0.61 }
  ],
  "personPrefs": [
    { "personId": 12, "personName": "诺兰",   "roleKind": "director", "score": 0.91 },
    { "personId": 27, "personName": "弗里曼", "roleKind": "actor",    "score": 0.75 }
  ],
  "avgRatingGiven": 7.4,
  "ratingCount": 12
}
```

UI：
- "最爱的类型 Top 5"水平条形图
- "最爱的导演/演员/编剧 Top 5"，按 `roleKind` 分 Tab
- 顶部小卡：平均打分 + 评分次数

### 数据怎么来
- 来源表：`user_genre_pref(user_id, genre_id, score)`、`user_person_pref(user_id, person_id, role_kind, score)`
- 计算职责：`UserFeatureBuilder`（Phase C），当 `moveme.recommendation.enabled = true` 时定时刷新
- 默认关闭时这两张表为空，前端会显示"暂无数据，先去评几部电影"
- 当下手动触发可走 SQL：

```sql
-- 把当前用户的高分电影对应的类型聚合（最简化口味）
INSERT INTO user_genre_pref (user_id, genre_id, score)
SELECT r.user_id, mg.genre_id, AVG(r.score) / 10.0
FROM ratings r
JOIN movie_genre mg ON r.movie_id = mg.movie_id
WHERE r.user_id = ? AND r.score >= 7
GROUP BY r.user_id, mg.genre_id
ON DUPLICATE KEY UPDATE score = VALUES(score);
```

完整算法见 [05-推荐算法详解](./05-推荐算法详解.md) 第 6 节。

## 7. 安全与隐私

- 所有 `/users/me/**` 接口都从 JWT 取 `userId`，不接受请求参数指定 `userId`（避免越权）
- 头像文件名固定为 `{userId}.{ext}`，新上传覆盖旧的，不会留垃圾
- 评分/收藏/历史均按 `(user_id, movie_id)` UNIQUE 约束防重（评分/收藏）或允许重复（历史）

## 8. 常见问题

| 现象 | 原因 / 处理 |
|---|---|
| 个人页空白 | token 失效。前端拦截器应自动 logout，重新登录 |
| 头像上传 413 | 超过 10MB。前端可先压缩或改 `application.yml` 的 `max-file-size` |
| 口味档案永远是空 | 推荐离线计算 `moveme.recommendation.enabled=false`；要么打开、要么手动跑 SQL 聚合 |
| 历史时间线时间错乱 | mysql 时区与 Java 时区不一致。jdbc URL 加 `serverTimezone=Asia/Shanghai` |
| 收藏取消后还显示在网格 | 前端未刷新列表。`removeFavorite` 后应 `await reload()` |
