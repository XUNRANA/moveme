# 34 - AI 推荐功能说明

> 基于小米 MiMO 大模型的 AI 电影推荐，支持**多轮聊天对话**和**一键推荐**两种模式。
>
> 需要登录使用。对话历史通过 localStorage 持久化，支持 Gemini 式 URL 路由（`/recommend/:id`）。

---

## 1. 功能概述

### 1.1 聊天模式

自由对话，可以问任何电影相关的问题。MiMO 会根据上下文回答，支持多轮对话。

特点：
- 普通聊天不注入用户画像，直接调用 MiMO API
- 如果回复中提及数据库中的电影，自动在电影名旁插入详情页链接
- 对话历史持久化到 localStorage，刷新不丢失

### 1.2 一键推荐

点击「一键推荐」按钮，系统自动将用户的观影档案（类型偏好、影人偏好、评分记录、收藏列表）注入 prompt，要求 LLM 从 TOP250 中挑选**一部**最适合的电影，用 3-5 段文字详细介绍（剧情、亮点、导演/演员、获奖），并附上详情页链接。

## 2. 架构设计

```
前端 Recommend.vue
  ├── 聊天模式 → POST /api/v1/recommend/chat
  └── 一键推荐 → POST /api/v1/recommend/quick

后端 RecommendController
  ├── /chat   → RecommendService.chat()           → 简单系统提示 → MiMO API
  ├── /quick  → RecommendService.quickRecommend()  → 片单 + 用户画像 → MiMO API
  └── /stream → RecommendService.chatStream()       → SSE 流式（备用）

MiMO API（OpenAI 兼容格式）← RestTemplate 调用
```

**为什么用后端代理而不是前端直调：** API Key 安全，不暴露给浏览器。

## 3. 后端模块

### 3.1 配置

**`application.yml`** 中 `moveme.llm.mimo` 节点：

```yaml
moveme:
  llm:
    mimo:
      api-key: ${MIMO_API_KEY:}          # 环境变量，必填
      model: mimo-v2.5-pro               # 默认模型
      base-url: https://token-plan-cn.xiaomimimo.com/v1  # Token Plan 专用
```

> **注意：** MiMO Token Plan 用户的 base URL 是 `token-plan-cn.xiaomimimo.com`，不是 `api.xiaomimimo.com`。

**`LlmProperties.java`** — `@ConfigurationProperties(prefix = "moveme.llm")`，内部类 `MimoProperties` 含 `apiKey`、`model`、`baseUrl`。

**`RestTemplateConfig.java`** — 注册 `RestTemplate` Bean，connect timeout 10s，read timeout 120s。

### 3.2 MimoService

`com.moveme.module.recommend.service.MimoService`

| 方法 | 说明 |
|---|---|
| `chat(List<Map<String,String>> messages)` | 非流式调用，返回完整响应字符串 |
| `chatStream(List<Map<String,String>> messages, StreamCallback callback)` | 流式调用（SSE），通过回调返回 token（当前前端未使用） |

请求体符合 OpenAI Chat Completions 格式：
```json
{
  "model": "mimo-v2.5-pro",
  "messages": [{"role": "system", "content": "..."}, {"role": "user", "content": "..."}],
  "temperature": 0.7
}
```

### 3.3 RecommendService

`com.moveme.module.recommend.service.RecommendService`

| 方法 | 系统提示 | 用户画像 | 片单注入 | 链接注入 |
|---|---|---|---|---|
| `chat()` | 简单助手提示 | 不注入 | 不注入 | 自动 |
| `quickRecommend()` | 完整规则 + 片单 | 注入 | 注入 top 300 | 自动 |
| `chatStream()` | 简单助手提示 | 不注入 | 不注入 | — |

**用户画像构建**（`buildUserContext`）：
- 类型偏好 top 5（`user_genre_pref`）
- 影人偏好 top 5（`user_person_pref`）
- 评分记录 top 15（含评论）
- 想看列表 top 10、已看列表 top 10

**电影链接注入**（`injectMovieLinks`）：
- 查询数据库所有电影的中文名和英文名
- 按片名长度降序匹配（避免短名截断长名）
- 跳过已是 markdown 链接格式的（`[片名](/...)`）
- 自动替换为 `[片名](/movies/{id})`

### 3.4 RecommendController

`com.moveme.module.recommend.controller.RecommendController` — `@RequestMapping("/api/v1/recommend")`

| 端点 | 方法 | 说明 |
|---|---|---|
| `POST /chat` | 非流式聊天 | 返回 `Result<String>` |
| `POST /chat/stream` | SSE 流式聊天 | 返回 `SseEmitter`（备用） |
| `POST /quick` | 一键推荐 | 返回 `Result<String>` |

所有端点都需要登录（JWT Bearer Token）。

### 3.5 关键文件清单

```
moveme-backend/src/main/java/com/moveme/
├── config/
│   └── RestTemplateConfig.java          # RestTemplate Bean
└── module/recommend/

```

## 4. 前端模块

### 4.1 Recommend.vue

页面布局：
- **左侧边栏**：对话历史列表（新建 / 切换 / 删除）
- **主区域**：Hero（空对话时）+ 消息列表 + 输入框

核心功能：
- **对话持久化**：localStorage（`moveme_conversations` + `moveme_active_conversation`）
- **Gemini 式 URL**：新建对话时导航到 `/recommend/{uuid}`，浏览器前进/后退可切换对话
- **打字机效果**：非流式 API 返回后，前端逐字显示模拟流式输出（15ms/字）
- **Markdown 渲染**：使用 `marked` 库，支持标题/列表/加粗/链接/代码
- **SPA 链接跳转**：点击 AI 回复中的 `/movies/xxx` 链接，拦截默认行为，用 `router.push` 跳转
- **响应式**：移动端侧边栏折叠，左下角浮动按钮切换

### 4.2 API 层

`api/movies.ts` 中新增：

```ts
interface ChatMessage { role: 'user' | 'assistant' | 'system'; content: string }

function chatRecommend(messages: ChatMessage[]): Promise<string>   // POST /recommend/chat
function quickRecommend(): Promise<string>                          // POST /recommend/quick
```

### 4.3 导航

`AppNavbar.vue` 中 "AI 推荐" 链接位于「发现」和「TOP250」之间，使用 `$route.path.startsWith('/recommend')` 判断激活状态。

## 5. 环境变量

| 变量 | 必填 | 说明 |
|---|---|---|
| `MIMO_API_KEY` | 是 | MiMO API Key，从 [Token Plan 页面](https://xiaoai.xiaomi.com/mimo/token-plan) 获取 |

设置方式（Windows PowerShell）：
```powershell
setx MIMO_API_KEY "your-api-key-here"
```

设置后需重启终端 / 后端进程才能生效。

---

## 6. 使用流程

1. 设置 `MIMO_API_KEY` 环境变量
2. 启动后端：`cd moveme-backend && ./mvnw spring-boot:run`
3. 启动前端：`cd moveme-frontend && npm run dev`
4. 登录后点击导航栏「AI 推荐」
5. 聊天模式：直接输入问题，如「推荐几部高分悬疑片」
6. 一键推荐：点击按钮，系统基于你的观影档案推荐一部电影

## 7. 设计决策

| 决策 | 选择 | 原因 |
|---|---|---|
| 流式 vs 非流式 | 非流式 + 前端打字机模拟 | Vite proxy 缓冲 SSE 导致流式不稳定 |
| 用户画像注入时机 | 仅一键推荐注入 | 聊天保持轻量，减少 token 消耗 |
| 对话持久化 | localStorage | 无需后端存储，用户数据留在浏览器 |
| 电影链接注入 | 后端正则替换 | 统一处理，不依赖 LLM 遵守链接格式 |
| 片单范围 | top 300 by 评分 | 平衡 prompt 长度和覆盖面 |
