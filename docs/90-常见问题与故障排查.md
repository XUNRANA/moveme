# 90 - 常见问题与故障排查

> 按现象分类。先看现象 → 找匹配的小节 → 跟着步骤排。

---

## 1. Docker / MySQL

### 1.1 `mysqld is not alive` 或连接被拒

**现象**：`docker exec moveme-mysql mysqladmin ping ...` 失败。

**排查**：
```bash
# 1) 容器在跑吗？
docker ps | grep moveme-mysql

# 2) 没在跑就启动
cd docker && docker compose up -d

# 3) 看 mysql 启动日志
docker logs moveme-mysql --tail 50
# 找 [ERROR] 行
```

**常见根因**：
- 数据卷损坏 → `docker compose down -v && docker compose up -d`（破坏性，会丢数据）
- 端口 3307 被占用 → 改 docker-compose.yml 端口映射
- 初始化 SQL 报错 → 看日志中 `01-schema.sql` / `02-seed-data.sql` 的 line N

### 1.2 SQL 初始化脚本不生效

**现象**：表没创建出来。

**关键事实**：MySQL 容器**只在数据卷为空时**执行 `init/*.sql`。如果数据卷有过历史数据，新 SQL 不会重跑。

**修复**：
```bash
docker compose -f docker/docker-compose.yml down -v   # -v 删数据卷
docker compose -f docker/docker-compose.yml up -d
```

### 1.3 中文乱码 / mojibake

**现象**：插入"剧情"，查出来是 `?? ?`。

**排查**：
```sql
SHOW VARIABLES LIKE 'character_set%';
-- 应该全是 utf8mb4
SHOW VARIABLES LIKE 'collation%';
-- 应该是 utf8mb4_unicode_ci
```

**修复**：
- `01-schema.sql` 顶部要有 `SET NAMES utf8mb4;` 和 `DEFAULT CHARACTER SET utf8mb4`
- 客户端连接串加 `?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai`
- 已抓到的乱码用 `TextEncodingRepairUtil` 修

### 1.4 外键约束失败

**现象**：`Cannot add or update a child row: a foreign key constraint fails`

**根因**：插入子表前，父表对应行还不存在。
- `movie_actor` 插入时 `person_id` 在 `persons` 里没有
- `movie_award.recipient_person_id` 没先 upsert person

**修复**：
- 导入流程严格按"先 person → 后 movie_actor"顺序
- 写代码时用 `upsertAllPersons()` 拿到 `Map<douban_person_id, persons.id>` 再插关联表

### 1.5 端口 3307 被占用

```bash
netstat -ano | grep :3307
# Windows: 看 PID，taskkill /F /PID <pid>
# Linux/Mac: lsof -i :3307
```

或者改 `docker/docker-compose.yml` 的端口映射 `"3308:3306"` + 同步改 `application.yml` 和 `.env`。

## 2. Spring Boot

### 2.1 `mvnw` 报"没有主清单属性"

`maven-wrapper.jar 3.2.0` 没 Main-Class，必须 `-cp` 启动：
```bash
java -Dmaven.multiModuleProjectDirectory=. -cp .mvn/wrapper/maven-wrapper.jar org.apache.maven.wrapper.MavenWrapperMain spring-boot:run
```

### 2.2 端口 8080 已被占用

```bash
# 找占用进程
netstat -ano | grep :8080
# 改 application.yml: server.port: 8081
# 同步改前端 vite.config.ts proxy target
```

### 2.3 启动卡在"Connecting to MySQL"

容器还没起来就启动后端。等一下：
```bash
docker exec moveme-mysql mysqladmin ping -uroot -proot123 --silent
# 输出 mysqld is alive 后再起后端
```

### 2.4 `@Transactional` 没生效

**现象**：异常被抛出但事务没回滚。

**根因（最常见）**：同类内部方法调用走不到 AOP 代理。

```java
class A {
    @Transactional public void outer() { inner(); }
    @Transactional public void inner() { ... }   // 不会开启新事务！
}
```

**修复**：
- 自注入 self proxy：`@Autowired private A self;` 然后 `self.inner()`
- 或者用 `TransactionTemplate` 显式提交
- 或者把 inner 拎到另一个 Service 里

### 2.5 MyBatis-Plus 字段不匹配

**现象**：`Unknown column 'createdAt' in 'field list'`

Java 驼峰 vs SQL 下划线没自动映射。

**修复**：`application.yml`:
```yaml
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
```

或在 entity 上 `@TableField("created_at")`。

### 2.6 JSON 列读写

**写**（Java → JSON）：
```java
@TableField(typeHandler = JacksonTypeHandler.class)
private Map<String, Double> genreVector;
```

或者类型用 `String`，自己 `objectMapper.writeValueAsString(map)` 后赋值。

**读**：同样用 typeHandler 反序列化。

## 3. JWT / 认证

### 3.1 401 Unauthorized

**排查**：
```bash
# Header 是不是带了？
curl -H "Authorization: Bearer eyJxxx..." http://localhost:8080/api/v1/me
```

- token 过期：access 2h，过期重新登录或用 refresh
- secret 不一致：`.env` 的 `JWT_SECRET` 必须 ≥ 256 bits
- SecurityConfig 路由没放行：检查 `permitAll()` 列表

### 3.2 CORS 错误

**现象**：浏览器 `Access-Control-Allow-Origin` 报错。

修 `WebMvcConfig.addCorsMappings(...)` 把 `http://localhost:5173` 加进 allowedOrigins。

### 3.3 Refresh token 失效

Redis 里 `auth:refresh_token:{userId}` 不存在或值不匹配。
- Redis 是不是没起？`docker exec moveme-redis redis-cli ping`
- TTL 过期？refresh 7 天，过期就只能重登

## 4. 前端 / Vite

### 4.1 接口 404 或代理失败

`vite.config.ts` 检查：
```typescript
proxy: {
  '/api': { target: 'http://localhost:8080', changeOrigin: true }
}
```

### 4.2 npm install 卡死

```bash
# 切淘宝镜像
npm config set registry https://registry.npmmirror.com
npm install
```

### 4.3 热重载不工作

- Vite 默认带 HMR，应该工作
- WSL2 / Docker mount 有时丢 fs 事件，加 `server.watch.usePolling: true`

## 5. 爬虫

### 5.1 豆瓣返回反爬页（"sec.douban.com"）

**现象**：HTML 是验证码页或重定向到 sec.douban.com。

**已实现的防护**：
- UA 轮换（`CrawlerConstants.USER_AGENTS`）
- 请求间隔（`CrawlerConstants.REQUEST_INTERVAL_MS` 默认 3-5 秒）
- 退避（连续失败时倍增等待时间）

**遇到了仍然挡**：
- 把间隔再调大（到 10s+）
- 加 cookie 模拟登录（豆瓣登录后限制宽松）
- 上代理池（暂未实现）

### 5.2 详情页解析失败

豆瓣偶尔 A/B 测试不同 HTML 结构。`DoubanMovieParser` 用 Jsoup CSS 选择器，写时挑稳定的 class 而不是 nth-child。

排查：把失败页面 HTML dump 下来本地试：
```java
String html = Files.readString(Paths.get("debug/failed.html"));
Document doc = Jsoup.parse(html);
// 在 IDE 里 evaluate
```

## 6. 推荐相关

### 6.1 movie_features 全空

Phase C 没跑过。手动触发：
```yaml
# application.yml
moveme:
  recommendation:
    enabled: true
    bootstrap-on-start: true   # 启动时跑一次
```

或 SQL 直接看：
```sql
SELECT COUNT(*) FROM movie_features;
SELECT * FROM movie_features LIMIT 1;
```

### 6.2 movie_similarity (CF_ITEM) 一直 0

**正常现象**：站内还没人评分，CF 没数据可算。等用户行为攒起来即可。

最低门槛：`cf-min-co-users` 默认 5 个共同评分用户的电影对。冷启动期可调到 1 临时观察。

### 6.3 推荐结果重复 / 缓存不刷新

`user_reco_cache` TTL 24h。修改了行为想立即看新结果：
```sql
DELETE FROM user_reco_cache WHERE user_id = ?;
```

或者实现 `RatingChangedEvent` 监听器主动 invalidate。

## 7. 种子导入（Phase B）

### 7.1 `Top250 seed import skipped: movies table already has X rows`

期望行为：`SeedAutoRunner` 检测 movies 表已有 ≥ 50 行，跳过。

**想强制重导**：
```sql
-- 顺序很重要，先关联表
TRUNCATE movie_genre; TRUNCATE movie_country; ...;
SET FOREIGN_KEY_CHECKS = 0;
TRUNCATE movies; TRUNCATE persons;
SET FOREIGN_KEY_CHECKS = 1;
-- 重启后端
```

### 7.2 `Cannot deserialize: Unrecognized field xxx`

DTO 字段名和 JSON 不一致。Jackson 默认严格。

修复：DTO 类加注解
```java
@JsonIgnoreProperties(ignoreUnknown = true)
public class SeedMovieDTO { ... }
```

或加：
```yaml
spring:
  jackson:
    deserialization:
      fail-on-unknown-properties: false
```

### 7.3 `Duplicate entry 'xxx' for key 'douban_id'`

幂等没做对。upsertMovie 必须先 selectByDoubanId 判断走 update 还是 insert，不要直接 insert。

## 8. 性能 / 慢查询

### 8.1 启用慢查询日志

```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 0.5;   -- 500ms
SHOW VARIABLES LIKE 'slow_query_log_file';
```

`tail -f` 那个文件看慢 SQL。

### 8.2 EXPLAIN 看执行计划

```sql
EXPLAIN SELECT * FROM movies WHERE year=1994 ORDER BY douban_rating DESC LIMIT 20;
-- 看 type 列：const/eq_ref/ref/range > index > ALL
-- type=ALL 全表扫，需要加索引
```

### 8.3 FULLTEXT 不命中

```sql
-- 必须用 MATCH ... AGAINST
SELECT * FROM movies
WHERE MATCH(title, title_cn, title_en, summary)
      AGAINST('救赎' IN NATURAL LANGUAGE MODE);

-- LIKE '%救赎%' 不会走 FULLTEXT
```

ngram 分词最小 token 默认 2。`SHOW VARIABLES LIKE 'ngram_token_size';`

## 9. 其它

### 9.1 `.env` 不生效

Spring Boot 不自动读 `.env`。两条路：
- 用 `docker compose` 时 `.env` 自动给 docker 用
- 用 IDE 启动时要么改 `application.yml`，要么用 `dotenv-spring` 库

### 9.2 时区不对

容器 mysql 默认 UTC，Java 默认本地。
- jdbc URL 加 `serverTimezone=Asia/Shanghai`
- mysql container env: `TZ: Asia/Shanghai`

### 9.3 海报 404

`PosterFileImporter` 拷到 `static/posters/`，确认 `WebMvcConfig.addResourceHandlers` 把 `/static/**` 映射出去：
```java
registry.addResourceHandler("/static/**")
        .addResourceLocations("file:./moveme-backend/static/");
```
